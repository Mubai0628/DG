# Capability Boundary Matrix v0.33

This v1 candidate matrix classifies each capability as enabled, approved-only,
runtime-only explicit, read-only, preview-only, simulation-only, disabled, or
deferred. It is a release-readiness contract, not a new execution surface.

## Status Categories

- Enabled: currently available in App or runtime with bounded behavior.
- Approved-only: requires explicit approval receipt and, where applicable, exact
  typed confirmation before mutation.
- Runtime-only explicit: available only through direct runtime/test fixtures or
  injected explicit opt-in helpers, not from the App Shell by default.
- Read-only: displays or returns summary data without mutation.
- Preview-only: builds summaries, proposals, or plans without execution.
- Simulation-only: exercises fixed fake/simulated flows without external effects.
- Disabled: intentionally unavailable.
- Deferred: planned for a future phase and blocked in v1 candidate.

## Candidate Gate

The v1 candidate can proceed only if all enabled and approved-only rows have
test evidence, manual QA coverage, summary-only EventStore behavior, and replay
behavior. Disabled and deferred rows must stay blocked by source boundary checks
and docs-lock assertions.

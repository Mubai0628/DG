workflow, capability boundaries, failure recovery, replay/audit, and release
artifacts. Evidence must be summary-only.

## Evidence Rules

- Store command names, status labels, counts, hashes, warning codes, and release
  labels only.
- Do not store raw prompt, source, diff, response, reasoning content, DOM, CSV,
  screenshot payload, clipboard content, file dialog content, API key,
  Authorization value, token, or secret.

# v0.10 Live Proposal Evaluation Post-Release Review

This review locks the v0.10 live proposal evaluation release line. P0N is
complete and should not continue expanding App execution.

The next phase is P0O: App-side Approved Execution MVP. P0O is allowed to open
only a narrow, human-approved, rollback-capable App-side user workspace apply
and rollback path.

## Release

- Tag: `v0.10.0-live-proposal-evaluation-rc.1`
- Title: Live proposal evaluation and golden cases, no App execution
- Status: released pre-release candidate

## Current Capabilities

- `web_table_to_csv` Convert remains the real conversion flow.
- Event Log / Replay remains available.
- Record Draft Event remains the App/Tauri local summary-event write path.
- P0I validation, approval draft, virtual apply, rollback checkpoint, and
  replay preview chains remain available.
- P0J sandboxed apply and rollback prototypes remain runtime/test scoped.
- P0K user workspace apply and rollback prototypes remain runtime-only and
  explicit fixture scoped.
- P0L proposal schema, fake harness, dry adapter, repair loop, App import, and
  chain preview remain available.
- P0M runtime-only live DeepSeek proposal adapter remains explicit opt-in with
  injected resolver and injected transport.
- P0N golden cases, offline evaluation, explicit opt-in live evaluation,
  failure metrics, App read-only evaluation summary, and telemetry redaction
  audit remain available.

## Disabled Capabilities

- App-side apply.
- App-side rollback.
- App-side apply/rollback event write.
- App approval execution.
- Production PermissionLease issuing.
- Git command execution.
- Shell command execution.
- Native bridge.
- Desktop action.
- App-side live DeepSeek execution for file writes.
- App API key read and App fetch/network.

## Release Hygiene

- No raw source is stored in release artifacts.
- No raw diff is stored in release artifacts.
- No raw CSV is stored in release artifacts.
- No raw prompt or raw response is stored in release artifacts.
- No reasoning_content is stored in release artifacts.
- No API key is stored in release artifacts.

## Boundary Lock

P0N proves proposal quality can be evaluated through golden cases, failure
taxonomy, repair metrics, and redaction audit without App execution.

P0O may introduce a narrow App-side approved execution MVP only after explicit
human approval, typed confirmation, path safety, checkpoint creation,
summary-only events, and rollback are in place. P0O must not introduce
auto-apply, model auto-execution, Git or shell execution, broad
PermissionLease, native bridge, desktop action, or raw content in events.

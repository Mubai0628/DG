# v0.11 App Approved Execution Post-Release Review

This review locks the v0.11 App-side approved execution release line. P0O is
complete and should not expand into arbitrary Git, shell, model, native bridge,
desktop action, or broad capability execution.

The next phase is P0P: Git / Shell Safe Lanes MVP. P0P may add only fixed
read-only Git summary lanes and fixed shell verification templates with
summary-only output and replay visibility.

## Release

- Tag: `v0.11.0-app-approved-execution-mvp-rc.1`
- Title: App-side approved apply and rollback MVP
- Status: released pre-release candidate

## Checks

- Scoped checks: passed during the P0O-008 RC polish.
- Stage-end full gates: passed during the P0O-008 RC polish.
- GitHub Actions: green for main and tag during the v0.11 release.
- Manual GUI QA: to be verified against the v0.11 manual QA checklist.

## Current Capabilities

- `web_table_to_csv` Convert remains the real conversion flow.
- Event Log / Replay remains available.
- Record Draft Event remains the App/Tauri local summary-event write path.
- App approved apply is available only through receipt, exact typed
  confirmation, path/content gates, checkpoint creation, and summary-only
  events.
- App approved rollback is available only from the matching private checkpoint
  after an approved apply.
- Checkpoint / preimage storage is local and private.
- Apply / rollback events are summary-only and replayable.
- Event Log / Replay shows approved apply count, approved rollback count, and
  latest approved execution summary.

## Disabled Capabilities

- Git execution.
- Shell execution.
- Arbitrary command execution.
- Git write commands.
- Shell install, network, or destructive commands.
- DeepSeek auto-execution.
- MCP/plugin/skills execution.
- Native bridge.
- Desktop action.
- Broad production PermissionLease issuing.
- Raw stdout/stderr in events.
- Raw source, raw diff, raw preimage, raw prompt, raw response, or API key in
  events.

## Known Warnings

- Tauri bundle id `.app` warning remains known.
- Vite chunk-size warning remains known.
- GitHub Actions Node deprecation annotation may appear when Actions forces a
  newer Node runtime.

## Release Hygiene

- No raw source is stored in release artifacts.
- No raw diff is stored in release artifacts.
- No raw CSV is stored in release artifacts.
- No raw prompt or raw response is stored in release artifacts.
- No checkpoint preimage is stored in event payloads.
- No API key is stored in release artifacts.
- Generated artifacts remain ignored.

## Boundary Lock

P0O proves a narrow App-side write path can exist only after human approval,
typed confirmation, strict path and content gates, private checkpoint creation,
summary-only events, and replay visibility.

P0P must preserve that boundary. It may add verification lanes after approved
apply / rollback, but those lanes must be fixed, non-arbitrary, summary-only,
redacted, replayable, and unable to write through Git or shell.

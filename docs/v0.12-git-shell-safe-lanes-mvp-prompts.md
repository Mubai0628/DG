# v0.12 — Git / Shell Safe Lanes MVP 阶段提示词

任务目标：完成所有小阶段任务。

本阶段目标是发布：

`v0.12.0-git-shell-safe-lanes-mvp-rc.1`

本阶段主题：

**Git / Shell Safe Lanes MVP**

v0.11 已经完成 App-side human-approved apply / rollback MVP。v0.12 的目标是在不引入任意命令执行的前提下，补齐“修改后验证”的最小闭环：

```text
approved apply / rollback
→ Git read-only summary lanes
→ Shell allowlist verification lanes
→ summary-only result events
→ replay / control projection
→ end-to-end smoke
```

本阶段仍然禁止任意 shell、任意 Git、Git 写命令、网络安装、native bridge、desktop action、DeepSeek auto-execution。

## 1. v0.12 最终目标

完成后应具备一条窄口径验证链：

```text
App-approved apply
→ Git status / diff summary / log / branch read lanes
→ Shell allowlist verification lanes
→ summary-only verification events
→ Event Log / Replay / Control Projection
→ approved rollback if needed
```

v0.12 应该允许：

- App 调用固定 Git read-only lanes。
- App 调用固定 shell verification templates。
- Git/Shell result 写 summary-only events。
- Event Log / Replay 显示 verification chain。
- E2E smoke 能证明“apply → verify → rollback”。

v0.12 仍然禁止：

- 任意 shell command。
- 任意 git command。
- Git write commands。
- shell install / network / destructive commands。
- raw stdout/stderr 持久化。
- raw diff/source/preimage/API key 进入 events。
- DeepSeek auto-execution。
- App 执行 MCP/plugin/skills/native bridge/desktop action。

---

## 2. 工作方式

每个小阶段必须：

1. 开始前检查 git 状态。
2. 只修改本小阶段相关文件。
3. 运行 scoped tests。
4. 本地 commit。
5. 不 push。
6. 不 tag。
7. 不创建 Release。

最终 RC 小阶段才允许：

1. 跑 full gates。
2. push main。
3. 等 GitHub Actions green。
4. tag。
5. push tag。
6. 创建 GitHub prerelease。

如果遇到阻塞：

- 停止。
- 不要绕过安全边界。
- 不要创建 half release。
- 给出 blocker report。

---

## 3. Git 工作流

每个小阶段开始时运行：

```powershell
git status --short
git status -sb
git log --oneline origin/main..HEAD
```

如果存在与当前小阶段无关的 dirty changes，停止并报告。

每个小阶段结束时运行：

```powershell
git diff --check
git diff --cached --check
```

然后只 stage 本阶段文件并 commit。

普通小阶段禁止：

```text
git push
git tag
gh release create
```

最终 release 小阶段才允许 push/tag/release。

普通小阶段只跑 scoped checks。

```text

```

# DW-P0P-003 — Git Read Lanes Tauri command, summary-only

## Goal

Add a fixed Tauri command for read-only Git lanes.

This is the first App-side real Git capability, but strictly read-only.

## Scope

Add:

- Tauri/Rust Git read lane command
- TypeScript wrapper
- App read-only summary panel / integration
- runtime/app docs
- tests

## Command name

Use a fixed allowlisted command name, for example:

```text
run_git_read_lane
```

If existing naming conventions require a different name, use that, but the name must clearly indicate read-only Git lane.

## Input

- `workspaceRoot`
- `workspaceRootRef`
- `lane: "status_summary" | "diff_summary" | "log_summary" | "branch_summary"`
- `pathspecs?`
- `maxOutputBytes?`
- `timeoutMs?`

## Workspace root safety

Block:

- missing root
- non-canonicalizable root
- drive root
- home/profile root
- system root
- system temp root itself
- symlink/junction/reparse root
- root outside user-chosen workspace

## Pathspec safety

Block:

- absolute path
- Windows drive path
- UNC path
- parent traversal
- null/newline
- shell metacharacters
- URL/query-like path
- `.git`
- `.env`
- `node_modules`
- `dist`
- `target`
- `.tmp`
- generated artifact path
- secret-like path

## Git execution boundary

Allowed fixed argv only:

- status summary:
  - `git status --porcelain=v1 --branch`
- diff summary:
  - `git diff --numstat --name-status -- <safe pathspecs>`
- log summary:
  - `git log --oneline --decorate -n <small fixed limit>`
- branch summary:
  - `git branch --show-current` and/or fixed read-only branch query

Prohibit:

- any Git write command
- arbitrary args
- shell command string
- shell interpreter
- env secret injection
- raw diff output

## Output summary

Return only:

- lane
- status
- branch summary
- file counts
- added/deleted line counts
- changed path summaries
- warnings
- command hash
- output hash
- truncated boolean
- no raw diff
- no raw stdout/stderr

## Event preview

Return summary-only event preview with `notWritten: true`.

Actual event wiring can happen in P0P-005.

## App wrapper

- `app/src/desktop-flow.ts`
- fixed wrapper only

## App UI


Badge:

```text

```

- optional safe pathspecs input
- button: `Run Git Read Lane`

## Tests

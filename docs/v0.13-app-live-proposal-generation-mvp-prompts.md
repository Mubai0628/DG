# v0.13 — App Live Proposal Generation MVP 阶段提示词

任务目标：完成所有小阶段任务。

本阶段目标是发布：

`v0.13.0-app-live-proposal-generation-mvp-rc.1`

阶段主题：

**App Live Proposal Generation MVP**

本阶段不是自动执行阶段。目标是让 App Shell 可以在显式用户确认、API key opt-in、请求边界、红线校验、redaction audit 全部成立后，发起一次 **live DeepSeek patch proposal generation**，并把结果导入既有 proposal / repair / validation / audit / approval / apply / rollback / replay 链。App 仍不能自动 apply、rollback、Git/shell、issue PermissionLease 或执行 desktop action。

---

## 全局当前状态

已发布：

- `v0.12.0-git-shell-safe-lanes-mvp-rc.1`
- `v0.13` 上一阶段准备任务将从 `v0.12 post-release review + P0Q roadmap lock` 开始。

当前重要能力边界：

- `web_table_to_csv` Convert 是真实 conversion flow。
- Record Draft Event 是 App/Tauri 本地 summary-event write path。
- App-side approved apply / rollback 已在 v0.11 引入，必须 human-approved、typed confirmation、path/content gates、checkpoint、summary events。
- Git / Shell Safe Lanes 已在 v0.12 引入，必须 fixed lanes / approved templates / summary-only outputs。
- Runtime live DeepSeek proposal adapter 已在 P0M 引入，但 runtime-only、explicit opt-in、injected resolver/transport。
- P0N 已有 live proposal evaluation / golden cases / telemetry audit。
- 当前 App Shell 仍不能：
  - live call DeepSeek
  - read raw API key
  - fetch/network directly
  - auto-apply
  - auto-rollback
  - Git write
  - arbitrary shell
  - issue broad PermissionLease
  - use native bridge / desktop action

# DW-P0Q-003 — Tauri Live DeepSeek Proposal Command, explicit opt-in, no apply

## 任务目标

新增固定 Tauri command，让 App 在显式 receipt 下可以发起 live DeepSeek proposal generation。

这是本阶段最高风险任务。必须保持最窄边界。

## 命令名

建议使用：

```text
generate_live_deepseek_patch_proposal
```

如项目已有命名约定，可使用一致命名，但必须固定、明确、allowlisted。

## 命令输入

必须包含：

- sessionReceipt
- apiKeySourceRef
- providerId: deepseek
- modelProfileId
- requestEnvelope
- objectiveSummary
- allowedPathRefs
- contextRefs
- maxResponseBytes
- timeoutMs

## API key 读取边界

- 只允许服务端/Tauri command 内部读取 allowlisted env ref。
- 禁止 App 传 raw API key。
- 禁止 command 返回 raw key。
- 禁止日志输出 key。
- 若无法安全读取 env，必须 fail closed。

## Network 边界

- 只允许此固定 command 发起 DeepSeek API 请求。
- 禁止 App fetch。
- 禁止 generic HTTP command。
- 请求必须 summary-only。
- 请求不得包含 raw source/raw diff/raw prompt dump/API key in payload。
- 不发送 tools/tool_choice。

## Response 边界

Command 可以返回：

- structured model response content as transient proposal candidate, or preferably validated summary if implemented。
- 必须限制 max response size。
- 必须不返回 API key。
- 必须不返回 raw prompt。
- 必须不返回 raw reasoning_content。
- reasoning_content 必须 dropped 或只记录 dropped flag。
- 返回给 App 后必须立即进入 P0L repair/schema validation。

## 禁止

- apply
- rollback
- EventStore write
- Git/shell
- PermissionLease
- native bridge
- desktop action

## App wrapper

新增/更新：

- `app/src/desktop-flow.ts`

只添加固定 wrapper，不添加 generic invoke。

## Tests

Rust/Tauri tests：

- missing receipt blocked
- wrong receipt blocked
- missing key ref blocked
- raw API key input blocked
- request with raw prompt/source/diff blocked
- response size over limit blocked
- no key in result
- no reasoning_content in result
- no apply/rollback/EventStore

App tests：

- fixed wrapper only
- no App fetch
- no API key input
- no generic invoke

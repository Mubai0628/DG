# v0.14 — End-to-End Coding Task MVP 阶段提示词

任务目标：完成所有小阶段任务。

本阶段目标是发布：

`v0.14.0-end-to-end-coding-task-mvp-rc.1`

主题：

**End-to-End DeepSeek Coding Task MVP**

本阶段不是继续增加大量底层能力，而是把 v0.11/v0.12/v0.13 已经完成的能力串成一个可重复、可回滚、可验证、可 replay 的端到端 coding task 流程：

```text
User objective
→ live DeepSeek proposal generation
→ repair / schema validation
→ model proposal import
→ chain integration
→ diff / validation / approval
→ human typed confirmation
→ approved apply
→ Git / shell verification safe lanes
→ summary events / replay
→ rollback if needed
```

本阶段必须继续保持：

```text
no auto-apply
no autonomous coding loop
no broad PermissionLease
no arbitrary Git/shell
no native bridge
no desktop action
no raw prompt/response/reasoning/API key in events
no raw source/diff/preimage in event payloads
```

---

## 0. 当前状态

当前已发布：

- `v0.13.0-app-live-proposal-generation-mvp-rc.1`
- Release title: `App live DeepSeek proposal generation MVP, no auto-apply`
- Release commit: `f8ca04a`

已完成：

- v0.1 Convert / Event Log / Replay 基线
- v0.11 App-side human-approved apply / rollback MVP
- v0.12 Git / Shell verification safe lanes MVP
- v0.13 App live DeepSeek proposal generation MVP

当前能力边界：

- App 可以显式 opt-in 请求 live DeepSeek 生成 patch proposal。
- Live proposal output 必须进入 repair / schema / import / chain preview。
- App approved apply / rollback 仍要求 human approval receipt 与 typed confirmation。
- Git / shell verification lanes 固定、bounded、summary-only。
- DeepSeek 不能 auto-apply。
- App 不能执行 arbitrary Git/shell。
- App 不能使用 native bridge / desktop action。

## 1. 工作方式

每个普通小阶段必须：

1. 开始前检查 git 状态。
2. 只修改本小阶段相关文件。
3. 运行 scoped tests。
4. 本地 commit。
5. 不 push。
6. 不 tag。
7. 不创建 GitHub Release。

最后 RC 小阶段必须：

1. 跑 scoped checks。
2. 跑 full gates。
3. push main。
4. 等 GitHub Actions main 绿。
5. tag。
6. push tag。
7. 等 tag GitHub Actions 绿。
8. 创建 GitHub prerelease。
9. 验证 release view。

如果遇到阻塞：

- 停止。
- 不要绕过安全边界。
- 不要创建 half release。
- 给出 blocker report。

每个小阶段开始时运行：

```powershell
git status --short
git status -sb
git log --oneline origin/main..HEAD
```

如果存在与当前小阶段无关的 dirty changes，停止并报告。

```powershell

```

普通小阶段禁止：

整个 v0.14 阶段禁止：

- DeepSeek auto-apply
- autonomous coding loop
- broad PermissionLease
- arbitrary Git/shell
- native bridge
- desktop action
- raw prompt / raw response / reasoning_content / API key 持久化

- rollback without checkpoint
- EventStore raw payload
- recursive delete
- directory delete
- symlink / junction / reparse traversal
- `.git`, `.env`, `node_modules`, `dist`, `target`, `.tmp` mutation

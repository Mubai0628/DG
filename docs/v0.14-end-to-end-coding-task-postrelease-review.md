# v0.14 End-to-End Coding Task Post-Release Review

This review closes the v0.14 End-to-End DeepSeek Coding Task MVP release line
and locks the next phase as P0S MVP hardening, recovery, and UX polish.

P0S must stabilize the existing approved execution flow. It must not expand the
App into auto-apply, autonomous coding, arbitrary Git or shell execution, broad
PermissionLease issuance, MCP/plugin/skills execution, native bridge, desktop
action, or raw payload persistence.

## Current Capabilities

- `web_table_to_csv` Convert remains the real baseline flow.
- Event Log / Replay remains available.
- Record Draft Event remains the App/Tauri local summary-event write path.
- App live proposal generation is available through explicit user opt-in only.
- Generated proposals enter repair, schema validation, model proposal import,
  and chain preview before they can enter any approved execution path.
- Human approval receipt and exact typed confirmation are required for approved
  apply and rollback.
- Approved apply is available through the narrow App/Tauri approved execution
  command path.
- Git and shell safe verification lanes are fixed, allowlisted, and
  summary-only.
- Rollback from checkpoint is available for approved execution flows.
- Summary-only events and replay reconstruct proposal, apply, verify, and
  rollback summaries.

## Disabled Capabilities

- Auto-apply.
- Autonomous coding loop.
- Arbitrary Git command execution.
- Arbitrary shell command execution.
- Broad PermissionLease issuance.
- MCP/plugin/skills execution.
- Native bridge.
- Desktop action.
- Model-driven file write without human approval receipt and typed
  confirmation.
- Raw prompt, raw response, reasoning_content, API key, raw source, raw diff,
  raw preimage, raw CSV, or raw DOM in events, logs, or UI summaries.

## P0S Boundary Lock

P0S may harden regression, stale snapshot handling, conflict handling, failure
recovery UX, replay/audit timeline projection, manual QA, release smoke, build
warnings, and boundary checks.

P0S must preserve the v0.14 execution boundary:

```text
User objective
-> explicit opt-in live proposal
-> repair/schema/import/chain preview
-> human approval receipt
-> exact typed confirmation
-> approved apply
-> fixed Git/shell verification safe lanes
-> checkpoint rollback if needed
-> summary-only events/replay
```

P0S must not introduce auto-apply, an autonomous coding loop, broad
PermissionLease issuance, arbitrary Git/shell execution, MCP/plugin/skills
runtime execution, native bridge, desktop action, or raw content persistence.

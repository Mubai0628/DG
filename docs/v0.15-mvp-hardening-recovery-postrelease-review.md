# v0.15 MVP Hardening / Recovery Post-Release Review

This review closes the v0.15 MVP hardening, recovery, and UX polish release
line and locks the next phase as P0T Production Memory / Project Knowledge MVP.

P0T may introduce workspace-local project knowledge persistence, but it must not
turn memory into an automatic execution path. Memory must stay summary-only,
human-reviewed, auditable, revocable, and replayable.

## Current Capabilities

- `web_table_to_csv` Convert remains the real baseline flow.
- Event Log / Replay remains available.
- Record Draft Event remains the App/Tauri local summary-event write path.
- Approved apply and rollback are available through the narrow human-approved
  App/Tauri command paths.
- Git / Shell Safe Lanes are fixed, allowlisted, and summary-only.
- App live proposal generation is available through explicit opt-in only.
- E2E task wizard and sequencer surfaces can project the approved execution
  chain.

  snapshots, conflict handling, failure recovery, replay, and release smoke.

## P0T Boundary Lock

P0T may add a workspace-local project knowledge store for summary-only
`policy`, `project_fact`, and `pitfall` entries. The lifecycle must remain:

```text
candidate
-> human review
-> commit
-> recall
-> revoke
-> expire
-> replay
```

P0T must preserve the existing approved execution boundary. Memory recall may
inform later task context, but it must not trigger apply, rollback, Git, shell,
PermissionLease issuance, native bridge, desktop action, or model-direct policy
writes.

# v0.15 — MVP Hardening / Recovery / UX Polish 阶段提示词

任务目标：完成所有小阶段任务。

目标版本：

`v0.15.0-mvp-hardening-recovery-rc.1`

阶段主题：

**MVP Hardening / Recovery / UX Polish**

v0.14 已达到 “个人可用 DeepSeek coding workbench MVP” 的 RC 口径。本阶段不追求扩大能力半径，而是把已存在的端到端 coding task flow 打磨到更稳定、更可恢复、更可回归。

本阶段重点：

```text
E2E golden regression
stale snapshot / conflict handling
apply / rollback failure recovery
event replay / audit timeline hardening
manual QA and release smoke hardening
packaging warning / boundary cleanup where safe
```

本阶段不是：

```text
不是新一轮 broad capability expansion
不是 autonomous coding loop
不是 MCP/plugin/skills runtime
不是 desktop operator
不是 arbitrary Git/shell
不是 broad PermissionLease
不是 model auto-apply
```

---

## 0. 当前状态

当前已发布：

- `v0.14.0-end-to-end-coding-task-mvp-rc.1`
- Title: `End-to-end DeepSeek coding task MVP`

已完成主链路：

```text
App live DeepSeek proposal generation
→ repair/schema/import/chain preview
→ human approval receipt + typed confirmation
→ approved apply
→ fixed Git/shell verification safe lanes
→ rollback from checkpoint
→ summary-only events/replay
```

- `web_table_to_csv` Convert 仍是真实 baseline flow。

- Raw source / raw diff / raw preimage / raw prompt / raw response / reasoning_content / API key 不得进入 events / logs / UI summaries。

每个普通小阶段：

1. 开始前检查 git 状态。
2. 只修改本小阶段相关文件。
3. 运行 scoped checks。
4. 本地 commit。
5. 不 push。

7. 等 tag CI green。
8. 创建 GitHub prerelease。
9. verify release view。

# DW-P0S-003 — Stale Snapshot / Conflict Handling Hardening

## Goal

Harden stale snapshot and conflict handling around approved apply / rollback.

## Scope

Runtime/Tauri/App as needed.

No new broad capability.

## Requirements

Approved apply must fail closed when:

- target file changed after approval
- expectedBeforeHash mismatch
- checkpoint cannot be created
- path becomes symlink/junction/reparse
- target disappears unexpectedly for update/delete
- create target already exists
- allowed path set changes
- receipt expired
- receipt hash mismatch
- validation/audit/proposal ids mismatch

Rollback must fail closed when:

- checkpoint missing
- checkpoint hash mismatch
- current file hash does not match applied hash
- rollback target became symlink/junction/reparse
- rollback would delete directory
- rollback would escape workspace

## UX

App should show:

- conflict detected
- stale snapshot
- revalidate required
- rollback unavailable / rollback safe
- nextAction

No auto-overwrite.

No auto-rollback unless explicitly already designed and user-approved. Prefer no auto-rollback.

## Tests

Rust/Tauri:

- stale update blocked
- stale delete blocked
- create conflict blocked
- checkpoint creation failure handled
- rollback current hash mismatch blocked

App:

- conflict state displays safe message
- apply disabled after stale conflict
- revalidate guidance appears
- no raw content displayed

## Commands

```powershell
pnpm app:typecheck
pnpm app:test
cargo check --manifest-path app/src-tauri/Cargo.toml
pnpm check:boundaries
pnpm check:secrets
git diff --check
git diff --cached --check
```

Run targeted Rust tests if project supports explicit test names.

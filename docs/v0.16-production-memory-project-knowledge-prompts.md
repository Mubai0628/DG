## 1. 本阶段最终目标

完成 v0.16 后，必须具备一条窄口径 Production Memory / Project Knowledge MVP：

```text
E2E task evidence / verification / approved execution summary
→ memory candidate
→ human review
→ commit project_fact / pitfall / policy
→ persistent project knowledge store
→ recall summary in later coding task
→ revoke / expire / audit
→ replay reconstructable memory state
```

本阶段必须仍然保持：

```text
no automatic memory commit
no model-direct policy write
no raw prompt/source/diff/API key in memory
no unreviewed long-term memory
no memory-triggered apply/rollback
no Git/shell expansion
no native bridge/desktop action
```

## 5. 全局硬边界

整个 v0.16 阶段禁止：

- 自动长期记忆 commit
- 模型直接写 policy memory
- tool/external source 直接写 policy memory
- raw prompt 写入 memory
- raw source 写入 memory
- raw diff 写入 memory
- raw CSV / DOM / screenshot 写入 memory
- API key / Authorization / Bearer / private key 写入 memory
- preimage / backup content 写入 memory
- stdout/stderr 原文写入 memory
- memory 自动触发 apply/rollback
- memory 自动触发 Git/shell
- broad PermissionLease
- MCP/plugin/skills runtime
- native bridge
- desktop action
- uncontrolled EventStore write
- arbitrary filesystem write

允许：

- summary-only memory candidate
- human-reviewed memory commit
- project-level persistent knowledge store under `.deepseek-workbench`
- revoke / expire / audit events
- summary-only recall into context
- policy memory only from human-approved trusted source
- project_fact memory with evidence refs
- pitfall memory with trigger / mitigation
- App read/review/commit/revoke UX under strict controls

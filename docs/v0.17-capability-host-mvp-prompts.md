# v0.17 — Capability Host MVP 阶段提示词

任务目标：完成所有小阶段任务。

本阶段目标版本：

`v0.17.0-capability-host-mvp-rc.1`

本阶段主题：

**MCP / Plugin / Skills Capability Host MVP**

核心目标是把外部能力生态引入项目的统一 Capability Broker / Control Plane，但只做到：

- descriptor-first
- read-only discovery/listing
- metadata scan
- risk classification
- approval/lease preview
- App read-only surface
- redaction / boundary audit

本阶段不允许打开任何外部能力执行。

---

## 0. 当前状态

当前已发布：

- `v0.16.0-production-memory-project-knowledge-rc.1`
- Title: `Production Memory / Project Knowledge MVP`

此前已完成：

- v0.11 App-side Approved Execution MVP
- v0.12 Git / Shell Safe Lanes MVP
- v0.13 App Live Proposal Generation MVP
- v0.14 End-to-End Coding Task MVP
- v0.15 MVP Hardening / Recovery / UX Polish
- v0.16 Production Memory / Project Knowledge MVP

当前重要能力边界：

- `web_table_to_csv` Convert 是真实 conversion flow。
- Record Draft Event 是 App/Tauri 本地 summary-event write path。
- App-side approved apply / rollback 已存在，但必须 human approval receipt + typed confirmation。
- Git / shell verification safe lanes 已存在，但固定 allowlist、summary-only。
- App live DeepSeek proposal generation 已存在，但 explicit opt-in，且不能 auto-apply。
- Project Knowledge store 已存在，但 human-reviewed、summary-only。
- 当前仍没有 MCP / plugin / skills runtime execution。
- 当前仍没有 native bridge / desktop action。
- 当前仍没有 broad PermissionLease。
- 当前仍没有 arbitrary external tool execution。

---

## 1. 本阶段最终目标

完成 v0.17 后，必须具备一条安全的外部能力 host MVP 预览链：

```text
external capability metadata / manifest
→ schema validation
→ read-only discovery/listing
→ package/skill metadata scan
→ Capability Broker descriptor mapping
→ risk / policy / lease preview
→ App read-only Capability Host surface
→ redaction / boundary audit
→ RC release
```

本阶段必须保持：

```text
no MCP tool invocation
no plugin code execution
no skill runtime execution
no external process launch
no network connection to MCP servers
no native bridge
no desktop action
no arbitrary shell/git
no file mutation from capability metadata
no PermissionLease issuing for external execution
```

---

## 2. 工作方式

每个子任务必须：

1. 开始前检查 git 状态。
2. 只修改本子任务相关文件。
3. 运行 scoped tests。
4. 本地 commit。
5. 不 push。
6. 不 tag。
7. 不创建 Release。

最终 RC 子任务才允许：

1. 跑 full gates。
2. push main。
3. 等 GitHub Actions green。
4. tag。
5. push tag。
6. 创建 GitHub pre-release。

如果遇到阻塞：

- 停止。
- 不绕过安全边界。
- 不创建 half release。
- 输出 blocker report。

---

## 3. Git 工作流

每个子任务开始时运行：

```powershell
git status --short
git status -sb
git log --oneline origin/main..HEAD
```

如果存在与当前子任务无关的 dirty changes，停止并报告。

每个子任务结束时：

```powershell
git diff --check
git diff --cached --check
```

然后只 stage 本任务文件并 commit。

普通子任务禁止：

```text
git push
git tag
gh release create
```

最终 release 子任务才允许 push/tag/release。

---

## 4. 测试策略

普通子任务只跑 scoped checks。

禁止普通子任务运行：

```text
pnpm test                       # 无过滤全量
pnpm verify:ci
pnpm release:smoke
pnpm app:build
browser-extension build/test
eval:web-table-to-csv
full conformance matrix
```

阶段收口 / release 子任务必须跑 full gates。

Runtime tests 必须优先使用显式 Vitest 文件路径，例如：

```powershell
pnpm exec vitest run runtime/test/<specific>.test.ts
```

不要使用会误跑全量 suite 的宽泛 filter。

---

## 5. 全局硬边界

整个 v0.17 阶段禁止：

- MCP tool invocation
- MCP stdio process launch
- MCP HTTP/SSE/WebSocket connection
- plugin code import / eval / execution
- skill runtime execution
- arbitrary external process launch
- native bridge
- desktop action
- broad PermissionLease
- arbitrary Git/shell
- auto-apply
- model-driven external tool execution
- network fetch from App or runtime capability host
- reading API keys for external capability execution
- writing EventStore events that imply external execution occurred
- writing raw secrets / raw schemas / raw tool args into events
- file mutation from external capability metadata

- missing sourceType
- unknown sourceType
- missing capabilities
- duplicate capabilityId
- unsafe capability id
- AUTO policy for external source
- raw command/execution fields
- secret marker
- raw args/prompt/source/diff/response fields
- URL with query secrets
- local path traversal
- risk downgrade attempt
- desktop/native/shell claims without DISABLED/MANUAL_ONLY
- broad wildcard scopes

## Validation should warn

- missing publisher summary
- missing version
- unknown category
- unknown operation kind
- network flag present
- filesystem flag present
- desktop flag present
- external source with high risk
- manifest lacks risk notes

## Output

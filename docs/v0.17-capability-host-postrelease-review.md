## Current Capabilities

- Convert: `web_table_to_csv` remains the real conversion flow.
- Event Log / Replay: summary-only event projection remains available.
- App-approved apply / rollback: human-approved, typed-confirmation gated, and
  rollbackable.
- Git / shell safe lanes: fixed / bounded / summary-only verification lanes.
- App live proposal generation: explicit opt-in and no auto-apply.
- Project Knowledge: human-reviewed and summary-only.
- Capability descriptor manifest: validates external capability descriptor
  metadata.
- MCP / plugin / skill metadata scanner: metadata-only preview.
- Capability Broker external descriptor preview: maps descriptors to risk,
  policy, approval, and lease preview summaries.
- App Capability Host read-only surface: previews MCP, plugin, and skill
  descriptors without connecting, installing, running, invoking, or issuing
  leases.
- External capability metadata redaction / boundary audit: blocks raw args,
  raw prompt/source/diff/response fields, secret markers, command fields,
  execution readiness, and issued lease claims.

## Disabled Capabilities

- MCP server connection
- MCP tool invocation
- MCP mutating tools
- plugin code execution
- skill runtime execution
- native bridge
- desktop action
- arbitrary Git/shell
- broad PermissionLease
- autonomous agent tool execution

## Release Hygiene

- Descriptor-first capability handling is preserved.
- External descriptors are summary-only.
- Raw prompt, raw response, raw source, raw diff, API key, and raw metadata
  payloads are not written to events.
- No external capability execution event is introduced.
- Generated build/eval artifacts remain ignored.

## Next Roadmap

P0V moves from descriptor-only preview to a narrow MCP read-only connection /
discovery MVP. P0V must preserve no MCP tool invocation, no mutating MCP
operation, no resource content read by default, no arbitrary process spawn, no
hidden App connection, and no external mutation.

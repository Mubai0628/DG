## Current Capabilities

- Convert: `web_table_to_csv` remains the real conversion flow.
- Event Log / Replay: summary-only event and replay projection remains
  available.
- App approved apply / rollback: human-approved, typed-confirmation gated,
  checkpointed, and rollbackable.
- Git/shell safe verification lanes: fixed, bounded, and summary-only.
- Project Knowledge: human-reviewed, revocable, expirable, and summary-only.
- Capability Host descriptors: MCP, plugin, and skill metadata can be previewed
  as descriptors without external execution.
- MCP read-only connection / discovery metadata: fixed-profile,
  typed-confirmation discovery can list MCP resources, prompts, and tools
  metadata summaries.
- Capability Broker MCP descriptor integration: MCP metadata enters the broker
  as disabled/read-only descriptors.
- App MCP read-only connection surface: previews metadata counts and disabled
  controls without invoking tools or reading resource content.
- MCP metadata redaction audit: blocks raw metadata, resource content, command
  fields, secret markers, execution claims, and event-write claims.

## Disabled Capabilities

- MCP `tools/call`
- MCP mutating tools
- MCP resource content read by default
- plugin code execution
- skill runtime execution
- native bridge
- desktop action
- arbitrary process spawn
- arbitrary shell command
- broad PermissionLease
- autonomous agent tool execution

## Release Hygiene

- MCP connection remains metadata-only.
- Tool invocation and resource content reads remain disabled.
- Raw metadata, raw prompt, raw response, raw source, raw diff, API key,
  stdout, and stderr are not written to App summaries or events.
- Capability Broker receives read-only descriptors, not invocation grants.
- Generated build, eval, conformance, and local workspace artifacts remain
  ignored.

## Next Roadmap

P0W moves from MCP read-only metadata discovery to MCP tool invocation proposal
and approval design. P0W must remain proposal-first and must not execute MCP
`tools/call`, mutating tools, plugin code, skill runtime code, arbitrary
processes, arbitrary shell commands, native bridge actions, desktop actions, or
broad PermissionLease grants.

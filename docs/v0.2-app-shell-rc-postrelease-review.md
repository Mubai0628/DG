## What Works

- `web_table_to_csv` Convert remains the active desktop workflow.
- Event Log / Replay shows summary event and draft information.
- Control Plane Projection derives read-only state from event summaries.
- Chat / Run Canvas is draft-only and local-state-only.
- Approval / Diff / Audit surfaces are read-only.
- Memory Inspector is read-only and not connected to persistence.
- Bridge Proposal Preview is disabled when no live bridge proposal exists.

## What Is Intentionally Disabled

- Real chat / LLM execution.
- Real run creation from the UI.
- Patch apply.
- Git execution.
- Shell execution.
- MCP, plugin, or skills runtime.
- Native bridge, `nativeMessaging`, localhost bridge, or extension auto-send.
- Desktop action.
- Memory persistence UI.
- Automatic Convert from any bridge proposal.

## Boundary Lock

The App Shell can display draft-only or read-only planning surfaces. It cannot
execute those plans. Future P0G work must preserve:

1. read-only or proposal-only first,
2. summary-only events,
3. explicit approval before any future side effect,
4. no raw prompt, raw source code, raw DOM, raw CSV, API key, authorization
   header, environment value, screenshot, or clipboard content in event
   payloads.

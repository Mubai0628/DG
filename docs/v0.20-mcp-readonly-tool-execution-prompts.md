# v0.20 MCP Read-only Tool Execution MVP 阶段提示词

任务目标：完成所有小阶段任务。

目标发布版本：

`v0.20.0-mcp-readonly-tool-execution-rc.1`

阶段主题：

**Controlled MCP Read-only Tool Execution MVP**

本阶段从 v0.19 的 **MCP tool invocation proposal / approval design, no execution** 推进到非常窄的 **read-only MCP tool execution**。
允许的执行范围仅限：

```text
MCP read-only tool metadata
→ read-only allowlist / invocation contract
→ input schema validation / risk classifier
→ explicit approval receipt + typed confirmation
→ fixed Tauri command
→ bounded runtime callTool wrapper
→ redacted summary-only result
→ summary-only event / replay
```

本阶段仍然禁止：

```text
mutating MCP tools
arbitrary MCP tool call
plugin code execution
skill runtime execution
desktop action
native bridge
broad PermissionLease
autonomous agent tool execution
arbitrary process spawn
arbitrary shell
raw MCP tool output in events or UI
raw MCP tool arguments in events
```

## 1. 全局硬边界

整个 v0.20 阶段必须保持：

```text
no mutating MCP tool
no arbitrary MCP tool call
no plugin code execution
no skill runtime execution
no desktop action
no native bridge
no broad PermissionLease
no autonomous agent tool execution
no arbitrary process spawn
no arbitrary shell
no Git execution beyond existing fixed safe lanes
no raw tool output in events
no raw arguments in events
no API key / Authorization / Bearer / secret persistence
no raw prompt / raw response / raw source / raw diff persistence
```

read-only MCP tool execution 只有在同时满足以下条件时才允许：

```text
tool is classified read-only
tool id is allowlisted
connection profile is fixed and approved
input schema validation passes
risk classifier says read-only / safe
approval receipt exists
typed confirmation matches
Tauri command is fixed / non-generic
runtime callTool wrapper is bounded
output is size-limited and redacted
summary-only event is written
replay can display result summary
```

- MCP tool execution starts with read-only tools only.
- Tool must be declared by discovered MCP metadata.

- Tool must have input schema validation.

- malicious MCP server metadata

- oversized output
- long-running tool call
- network exfiltration by server
- resource content leakage
- approval bypass
- replay tampering
- event raw output leakage
- App hidden invocation
- cross-workspace profile confusion

## Implementation gate must be testable

Checklist categories:

- connection/profile safety
- allowlist safety
- schema validation safety
- risk classifier safety
- approval safety
- call boundary safety

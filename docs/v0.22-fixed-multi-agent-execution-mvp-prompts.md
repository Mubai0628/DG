# v0.22 Fixed Multi-Agent Execution MVP 阶段提示词

任务目标：完成所有小阶段任务。

本阶段目标是发布：

`v0.22.0-fixed-multi-agent-execution-mvp-rc.1`

阶段主题：

**Fixed Multi-Agent Execution MVP**

本阶段要把早期 Agent Dossier / Static Router / Agent Route Preview 推进到固定角色、固定路线、可审计、可回放的多 Agent 执行闭环。

## 1. 本阶段最终目标

完成 v0.22 后，必须具备一条固定多 Agent coding task chain：

```text
Task objective / run draft
→ Orchestrator creates fixed run plan
→ Coder produces proposal through existing proposal chain
→ Reviewer validates/audits proposal and risk
→ Verifier runs fixed verification lanes / evidence checks
→ Human approval remains required for apply/rollback
→ Summary-only agent events and replay timeline
```

本阶段允许：

- 固定角色：
  - orchestrator
  - coder
  - reviewer
  - verifier
- 固定路线：
  - `code_change`: orchestrator → coder → reviewer → verifier
  - `code_review`: orchestrator → reviewer → verifier
  - `documentation`: orchestrator → coder → reviewer
  - `verification`: orchestrator → verifier
- summary-only dossier handoff
- existing live proposal / import / validation / apply / verification chain reuse
- App read-only / controlled multi-agent run surface
- agent events / replay / audit timeline

本阶段禁止：

- dynamic agent bidding
- arbitrary agent creation
- agent-to-agent hidden raw prompt sharing
- agent direct file writes
- agent direct Git/shell execution
- agent direct MCP mutating tool invocation
- plugin/skill arbitrary runtime execution
- desktop action
- native bridge
- auto-apply
- broad PermissionLease

## Release notes must include

Current working flow:

```text
web_table_to_csv Convert remains the real conversion flow.
App-side approved execution remains human-approved and rollbackable.
Git/shell verification safe lanes remain fixed and summary-only.
MCP read-only tool execution remains approved/read-only/bounded.
Plugin/skill metadata sandbox remains no arbitrary execution.
Fixed multi-agent run can plan and replay orchestrator/coder/reviewer/verifier routes.
Agents cannot dynamically bid, auto-apply, or directly execute tools.
```

Non-goals:

```text
no dynamic agent bidding
no arbitrary agent creation
no autonomous arbitrary tool execution
no agent direct apply/rollback
no arbitrary Git/shell
no mutating MCP tools
no arbitrary plugin/skill runtime
no native bridge
no desktop action
no broad PermissionLease
no raw prompt/source/diff in agent events
```

Safety:

```text
fixed roles
fixed routes
summary-only handoff dossier
Capability Broker gated planning
human approval for apply
typed confirmation
summary-only events
replay timeline
```

## Push / tag / release

After commit and clean tree:

```powershell
git status --short
git log --oneline origin/main..HEAD
```

Push main:

```powershell
git push origin main
```

Wait for GitHub Actions main to be green.

Tag:

```powershell
git tag v0.22.0-fixed-multi-agent-execution-mvp-rc.1
git push origin v0.22.0-fixed-multi-agent-execution-mvp-rc.1
```

Wait for tag Actions to be green.

Create GitHub pre-release:

```powershell
gh release create v0.22.0-fixed-multi-agent-execution-mvp-rc.1 `
```

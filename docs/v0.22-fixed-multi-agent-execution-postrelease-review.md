## Current Capabilities

- Convert: `web_table_to_csv` remains the real conversion flow.
- Event Log / Replay: summary-only event replay remains available.
- Record Draft Event: App/Tauri summary draft-event write path remains
  explicit and local.
- App approved execution: apply and rollback are human-approved,
  typed-confirmation, path/content/checkpoint gated.
- Git/shell safe lanes: fixed allowlists, verification-only, summary-only.
- Production project knowledge: human-reviewed, summary-only, replayable,
  revocable, and expirable.
- MCP read-only tools: approved, allowlisted, bounded, summary-only.
- Plugin/skill metadata sandbox: manifest and metadata governance only; no
  arbitrary plugin or skill runtime.
- Fixed multi-agent execution: fixed orchestrator/coder/reviewer/verifier route,
  summary-only handoff dossiers, capability planning, App preview, and replay.

## Release Hygiene

- No raw prompt, source, diff, model response, API key, raw screenshot, raw OCR,
  raw stdout, or raw stderr is introduced by the fixed multi-agent release.
- Generated directories such as `.tmp/`, `dist/`, `target/`, and `node_modules/`
  remained ignored.
- P1A must not broaden App execution. Desktop Observer work starts as
  observation-only, user-triggered, summary-only metadata.

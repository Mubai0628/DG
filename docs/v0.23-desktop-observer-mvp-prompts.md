# v0.23 — Desktop Observer MVP 阶段提示词

任务目标：完成所有小阶段任务。

本阶段目标版本：

`v0.23.0-desktop-observer-mvp-rc.1`

本阶段主题：

**Desktop Observer MVP, no action**

本阶段只打开桌面观察能力，不打开桌面动作能力。目标是让 App 可以在用户明确触发下读取有限、可审计、summary-only 的桌面窗口 / 应用 / 显示器 / 可选截图元数据，并把这些观察结果作为 context evidence / agent dossier refs。不得点击、输入、选择、写剪贴板、操作文件对话框、远控窗口、录屏或后台隐藏采集。

## 1. 本阶段最终目标

完成 v0.23 后，必须具备一条窄口径 Desktop Observer MVP：

```text
User explicitly requests desktop observation
→ fixed Tauri observer command runs
→ returns foreground/window/app/display metadata summary
→ optional screenshot metadata/redaction boundary is represented, no raw screenshot persisted by default
→ App read-only Desktop Observer surface displays summary
→ observation summary becomes evidence refs for Context / Agent Dossier
→ privacy/redaction audit validates no raw screenshot/secret/clipboard/prompt leakage
→ replay/audit summary can show observation refs
```

本阶段必须仍然保持：

```text
no desktop action
no click/type/select
no clipboard write
no file dialog automation
no native bridge broad action
no remote control
no screen recording
no hidden background capture
no dynamic agent desktop control
no raw screenshot persistence by default
no raw OCR text persistence by default
no API key / secret / raw prompt leakage
```

## ADR must decide

- Desktop Observer is observation-only.
- Observation must be user-triggered.
- No background hidden capture.
- No desktop action.
- No click/type/select.
- No clipboard write/read by default.
- No file dialog automation.
- No raw screenshot persistence by default.
- No raw OCR text persistence by default.
- Observation outputs are summary-only metadata / redacted evidence refs.
- Any screenshot support must be metadata / hash / redaction boundary first.
- Observation evidence may enter context / agent dossier only as summary refs.

- private window titles
- browser tabs
- usernames / emails in window title
- visible secrets
- screen capture leakage
- clipboard leakage
- OCR leakage
- background capture
- focus spoofing
- window handle mismatch
- multi-monitor privacy
- remote desktop / VM / browser profile leakage
- event/replay raw screenshot leakage
- agent misuse
- prompt injection from visible screen text
- desktop observation as covert data exfiltration

## Implementation gate must be testable

Checklist categories:

- profile safety
- user-trigger safety
- Tauri command allowlist
- window metadata safety
- screenshot/redaction safety
- event/replay safety
- App UI safety
- context/agent evidence safety
- CI/boundary safety

# v0.23 Desktop Observer Post-release Review

Status: post-release review for `v0.23.0-desktop-observer-mvp-rc.1`.

Release title: `Desktop Observer MVP, no desktop action`.

Release URL:
`https://github.com/Mubai0628/DG/releases/tag/v0.23.0-desktop-observer-mvp-rc.1`

## Release Summary

v0.23 completed the Desktop Observer MVP. The App Shell can surface
user-triggered, metadata-only desktop observation summaries and route summary
evidence refs into context and agent dossiers. The release deliberately keeps
the boundary below desktop control: no desktop action, no click/type/select, no
clipboard write, no file dialog automation, no hidden background capture, no
screen recording persistence, and no native bridge broad action.

## Current Capabilities

- Convert remains the real `web_table_to_csv` conversion flow.
- Event Log / Replay remains available for summary events and replay
  projections.
- Record Draft Event remains the App/Tauri local summary-event write path.
- Approved apply / rollback remains human-approved and rollbackable.
- Git / shell verification safe lanes remain fixed and summary-only.
- Production memory / project knowledge remains summary-only and replayable.
- Capability host descriptors remain governance metadata, not broad execution.
- MCP read-only connections and controlled read-only MCP tools remain bounded.
- Plugin / skill metadata sandboxing remains descriptor-first and non-mutating.
- Fixed multi-agent execution remains fixed-route with no dynamic bidding.
- Desktop Observer metadata-only support covers foreground/window/app/display
  summaries.
- Desktop Observer evidence refs can enter Context Assembly and Agent Dossiers
  as summary-only refs.

## Disabled Capabilities

- Desktop action.
- Click/type/select.
- Drag/drop or hotkey execution.
- Clipboard write.
- File dialog automation.
- Raw screenshot persistence by default.
- Raw OCR text persistence by default.
- Hidden background capture.
- Screen recording persistence.
- Native bridge broad action.
- Remote control.
- Autonomous desktop agent.
- Dynamic agent desktop control.
- App-side desktop action execution.

## Known Warnings

- Tauri may warn that the bundle identifier ends with `.app`.
- Vite may warn about large chunks during App build.
- GitHub Actions may report transient API or network delays when queried from a
  local shell; release status should be verified from the GitHub release page
  or `gh release view`.

## Release Hygiene

- No raw screenshot is persisted by default.
- No raw OCR text is persisted by default.
- No raw prompt, raw source, raw diff, raw response, API key, clipboard content,
  or secret marker is included in Desktop Observer summaries.
- Generated local build and eval artifacts remain ignored.
- Manual GUI QA is tracked by `docs/desktop-observer-manual-qa.md`; do not
  treat the checklist as evidence that manual QA has been performed.

## Next Phase

P1B moves from observation to proposal modeling only: Desktop Action Proposal
MVP, no execution. P1B can model future desktop action proposals, target
metadata validation, risk classification, approval drafts, dry-run simulation,
Capability Broker planning refs, App read-only display, and privacy audit.
Actual desktop action execution remains deferred.

# v0.25 Approved Desktop Action Execution MVP 阶段提示词

任务目标：完成所有小阶段任务。

目标版本：`v0.25.0-approved-desktop-action-execution-mvp-rc.1`

阶段主题：**Approved Desktop Action Execution MVP**

本阶段目标是从 v0.24 的 Desktop Action Proposal MVP 推进到一条极窄、显式审批、可审计、可回放的桌面动作执行路径。v0.25 只允许固定 allowlist 的低风险桌面动作，优先支持 `focus_observed_window` / `raise_observed_window` / `activate_observed_window` 这类窗口焦点类动作；仍然禁止任意 click/type/select/drag/drop、clipboard write、file dialog automation、screen recording、hidden background capture、remote control、broad native bridge、dynamic agent desktop control。

---

## 全局阶段边界

整个 v0.25 阶段必须始终保持：

```text
no arbitrary desktop action
no click/type/select/drag/drop execution
no clipboard write
no file dialog automation
no screen recording
no hidden background capture
no remote control
no broad native bridge
no dynamic agent desktop control
no autonomous desktop agent
no arbitrary Git/shell
no mutating MCP tools
no arbitrary plugin/skill runtime
no broad PermissionLease
no raw screenshot persistence
no raw OCR persistence
no raw prompt/source/diff/API key/event payload leakage
```

允许的第一批 approved desktop action 必须极窄：

```text
focus_observed_window
raise_observed_window
activate_observed_window
```

如果当前平台无法安全实现外部窗口 focus/raise/activate，则命令必须 fail closed 或返回 summary-only `unsupported_platform`，不得降级为任意 native bridge 或 shell。

所有真实桌面动作必须满足：

```text
Desktop Observer evidence exists
Desktop Action Proposal exists
target/window/app metadata validation passes
risk classifier allows the action
approval receipt exists
typed confirmation exact match
target evidence is not stale
fixed Tauri command only
summary-only event
replay can reconstruct action summary
privacy/redaction audit passes
```

普通子任务只跑 scoped checks；只有最终 RC 子任务跑 full gates、push、tag、release。

每个子任务都必须本地 commit；普通子任务禁止 push/tag/release。

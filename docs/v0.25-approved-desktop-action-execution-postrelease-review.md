# v0.25 Approved Desktop Action Execution Post-release Review

Release tag:

```text
v0.25.0-approved-desktop-action-execution-mvp-rc.1
```

Release title:

```text
Approved desktop action execution MVP, narrow focus actions only
```

Status: published as a prerelease.

## Current Capabilities

- `web_table_to_csv` Convert remains the real conversion flow.
- approved apply / rollback exists behind human approval and typed confirmation.
- Git/shell verification safe lanes exist as fixed allowlisted, summary-only
  verification lanes.
- MCP read-only tool execution exists only for approved, read-only, bounded
  calls.
- plugin/skill manifest sandbox exists without arbitrary runtime execution.
- Fixed multi-agent execution exists without dynamic bidding.
- Desktop Observer provides metadata-only evidence.
- Desktop Action Proposal models future desktop actions without execution.
- Approved Desktop Action Execution allows only narrow observed-window actions:
  - `focus_observed_window`
  - `raise_observed_window`
  - `activate_observed_window`

## Disabled Capabilities

- Click/type/select execution.
- Clipboard write.
- File dialog automation.
- Drag/drop.
- Screen recording.
- Hidden capture.
- Remote control.
- Broad native bridge.
- Dynamic agent desktop control.
- Autonomous desktop agent.

## Release Hygiene

- Desktop action results are summary-only.
- Event and replay outputs do not contain raw screenshots, OCR text, clipboard
  content, raw prompts, raw source, raw diffs, or API keys.
- Replay does not re-execute desktop actions.
- Unsupported desktop action execution paths fail closed or return
  `unsupported_platform`.

## Known Limitations

- The App cannot execute click/type/select, clipboard write, file dialog, drag
  and drop, or arbitrary desktop actions.
- v0.25 does not provide broad native bridge or remote control.
- v0.25 does not let dynamic agents control the desktop.

## Next Stage

P1D expands desktop action proposal modeling only. It adds proposal schemas,
freshness checks, risk classification, sequence simulation, read-only App
display, and privacy/redaction audit without enabling click/type/clipboard or
file dialog execution.

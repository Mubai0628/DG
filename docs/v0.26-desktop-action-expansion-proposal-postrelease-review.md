# v0.26 Desktop Action Expansion Proposal Post-release Review

Release tag:

```text
v0.26.0-desktop-action-expansion-proposal-rc.1
```

Release title:

```text
Desktop action expansion proposals, no click/type execution
```

Status: published as a prerelease.

## Current Capabilities

- `web_table_to_csv` Convert remains the real conversion flow.
- Event Log / Replay remains summary-only and deterministic.
- Record Draft Event remains the App/Tauri local summary-event write path.
- Desktop Observer provides metadata-only foreground/window/app/display evidence.
- Desktop Action Proposal remains proposal-first and does not execute actions.
- Runtime desktop action expansion proposal schema covers future click, type,
  select, keyboard shortcut, clipboard, file dialog, drag/drop, scroll, and wait
  action proposals.
- Target freshness and screen mismatch checks produce summary-only freshness
  status.
- Sequence simulation models proposed action effects without desktop execution.
- Sensitive/destructive UI risk classifier escalates or blocks dangerous target
  summaries.
- App expanded desktop action proposal surface is read-only and disabled for
  execution.
- Runtime desktop action expansion redaction audit blocks raw screenshot, raw
  OCR, raw target text, clipboard content, file dialog raw paths, API keys, and
  execution fields.

## Disabled Capabilities

- Real click.
- Real type.
- Real select.
- Clipboard write.
- File dialog automation.
- Drag/drop.
- Hidden/background action.
- Remote control.
- Broad native bridge.
- Dynamic agent desktop control.
- Autonomous desktop agent.
- Screen recording.
- Raw screenshot persistence.
- Raw OCR persistence.

## Release Hygiene

- Expanded desktop action proposal artifacts are summary-only.
- Events and replay do not contain raw screenshots, OCR text, clipboard content,
  raw prompts, raw source, raw diffs, or API keys.
- Replay does not execute or re-execute desktop actions.
- v0.26 preserves the v0.25 narrow approved observed-window lane and does not
  open click/type/select execution.

## Known Limitations

- The App cannot execute click/type/select, clipboard write, file dialog,
  drag/drop, or arbitrary desktop actions.
- v0.26 does not provide broad native bridge, remote control, or autonomous
  desktop agent behavior.
- Expanded action readiness is a proposal/audit signal only; it is not execution
  authority.

## Next Stage

P1E opens only a narrow approved expanded desktop action lane: one safe click on
an observed safe target or one safe type into an observed safe text field. Both
lanes require observer evidence, proposal summaries, target validation,
freshness and mismatch checks, risk classification, simulation, approval
receipt, typed confirmation, a fixed Tauri command, and summary-only events.

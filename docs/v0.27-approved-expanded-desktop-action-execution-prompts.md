# v0.27 Approved Expanded Desktop Action Execution 阶段提示词

任务目标：完成所有小阶段任务。

阶段目标：完成 `v0.27.0-approved-expanded-desktop-action-execution-rc.1`。

本阶段主题：**Approved Expanded Desktop Action Execution, narrow click/type only**。

本阶段只允许把 v0.26 已有的 expanded desktop action proposals 中极窄的一部分推进到受控执行：

- 单次 click：只能作用于观察到的 safe target。
- 单次 type：只能作用于观察到的 safe text input target。
- 必须 human approval。
- 必须 typed confirmation。
- 必须 freshness / screen mismatch check。
- 必须 fixed Tauri command。
- 必须 summary-only events / replay。

本阶段仍然禁止：

- clipboard write
- file dialog automation
- drag/drop
- multi-step desktop automation
- background / hidden desktop action
- remote control
- broad native bridge
- dynamic agent desktop control
- autonomous desktop agent
- arbitrary click/type
- screen recording
- raw screenshot persistence
- raw OCR persistence
- App-side broad desktop execution

# DW-P1E-005 — Tauri Fixed Expanded Desktop Action Command, click/type only

## Goal

新增固定 Tauri command 执行 approved click/type。必须平台安全，unsupported platform 需 safe result。

## Command

固定命令名建议：

```text
execute_approved_expanded_desktop_action
```

不得新增 generic native bridge。

## Input

- `contract`
- `receipt`
- `actionKind`
- `targetRef`
- `windowRef`
- `appRef`
- `displayRef`
- `boundsSummary`
- `textPayload?` for type only

## Behavior

- Validate receipt.
- Validate contract.
- Validate freshness.
- Validate target metadata.
- On unsupported platform / permissions missing, return `unsupported_platform` or `permission_required`, summary-only.
- If supported, execute only:
  - single click target center or safe point inside target bounds
  - type text into active observed target after focus validation
- No clipboard write.
- No file dialog.
- No drag/drop.
- No multi-step sequence.
- No hidden/background action.
- No remote control.

## Output

Summary-only:

- `actionExecutionId`
- `actionKind`
- `status: executed | blocked | unsupported_platform | permission_required`
- `targetRef`
- `windowRef`
- `appRef`
- `boundsHash`
- `textLength?`
- `textHash?`
- warning codes
- result hash
- eventPreview `notWritten: true`

No raw screenshot, OCR, target text, or text payload.

## Tests

Rust/Tauri tests where practical:

- unsupported platform safe result
- missing receipt blocked
- wrong typed confirmation blocked
- unsupported action kind blocked
- clipboard/file dialog/drag/drop blocked
- raw text leak blocked
- summary-only output

App tests:

- fixed wrapper only
- no generic native bridge
- no enabled command unless receipt + contract present

# v0.27 Approved Expanded Desktop Action Post-release Review

Release tag:

```text
v0.27.0-approved-expanded-desktop-action-execution-rc.1
```

Release title:

```text
Approved expanded desktop actions, narrow click/type only
```

Status: published as a prerelease.

## Current Capabilities

- `web_table_to_csv` Convert remains the real conversion flow.
- Event Log / Replay remains summary-only and deterministic.
- App live proposal generation remains explicit opt-in and gated.
- Approved workspace apply and rollback remain human-approved, typed-confirmed,
  checkpointed, and rollbackable.
- Git / shell safe verification lanes remain fixed, bounded, and summary-only.
- Project Knowledge can provide reviewed summary refs.
- MCP read-only discovery and read-only tool execution can provide bounded
  summary evidence.
- Plugin / Skill metadata governance can validate manifests, metadata, sandbox
  summaries, and simulations without arbitrary runtime execution.
- Fixed multi-agent execution can coordinate fixed orchestrator, coder,
  reviewer, and verifier summaries.
- Desktop Observer provides metadata-only foreground/window/app/display
  evidence.
- Desktop Action Proposal remains proposal-first with risk and simulation
  summaries.
- Approved desktop action execution supports the existing narrow
  focus/raise/activate observed-window lane.
- Approved expanded desktop action execution supports only single safe click
  and single safe type lanes after explicit approval, typed confirmation,
  target validation, freshness checks, and fixed command execution.

## Disabled Capabilities

- Broad desktop action.
- Clipboard write.
- File dialog automation.
- Drag/drop.
- Background/hidden action.
- Remote control.
- Dynamic agent desktop control.
- Arbitrary Git/shell.
- Mutating MCP tools.
- Arbitrary plugin/skill runtime.
- Broad native bridge.
- Autonomous desktop agent.
- Replay re-execution.

## Release Hygiene

- Approved expanded desktop action events are summary-only.
- Replay surfaces do not re-execute desktop actions.
- Raw screenshot, raw OCR, raw target text, raw prompt, raw response, raw
  source, raw diff, and API keys are not persisted in events, summaries, or App
  state snapshots.
- Unsupported desktop platforms return safe summary results.
- Generated build outputs remained ignored during the RC.
- GitHub Actions main and tag runs were green for the RC tag.

## Known Warnings

- GitHub Actions reports a Node.js 20 deprecation annotation for upstream
  actions forced onto Node.js 24.
- Tauri build reports the known bundle identifier warning.
- Vite reports the known large chunk-size warning.

## Next Stage

P1F moves from isolated safe lanes to a cross-surface agent workflow RC:
scenario schema, fixed planner, App composer, evidence integration, approved
actions sequencer, unified replay/audit timeline, smoke fixture, manual QA, and
release polish. P1F does not expand execution authority; it only integrates,
demonstrates, audits, replays, and verifies existing controlled lanes.

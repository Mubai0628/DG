## Current Capabilities

- Convert remains the real `web_table_to_csv` conversion flow.
- Approved execution / rollback remains bounded by existing receipts,
  confirmations, path checks, and summary events.
- Git/shell verification safe lanes remain fixed-template and summary-only.
- Project knowledge recall remains summary-only.
- MCP read-only tool execution can provide evidence summaries.
- Plugin/skill metadata governance remains descriptor and metadata based.
- Fixed multi-agent routing remains static; no dynamic bidding is enabled.
- Desktop observer remains metadata/evidence oriented.
- Desktop action proposal remains proposal-first.
- Approved narrow and expanded desktop actions remain bounded to existing
  approved lanes.
- Cross-surface workflow planner, preview, approved sequencer, evidence
  summary, and replay/audit timeline are available for the North Star demo.

## Disabled Capabilities

- Dynamic agent bidding is disabled.
- Autonomous arbitrary tool use is disabled.
- Mutating MCP tools are disabled.
- Arbitrary plugin/skill runtime execution is disabled.
- Broad desktop actions are disabled.
- Clipboard and file dialog automation are disabled.
- Broad native bridge access is disabled.
- Arbitrary Git/shell execution is disabled.
- Auto-apply is disabled.

## Release Hygiene

- Replay and events remain summary-only.
- Raw prompt, source, diff, response, reasoning content, and API keys are not
  allowed in events, replay, or telemetry.
- Generated artifacts remain ignored.

## Follow-up

P1G should harden the North Star demo without widening execution scope. The
next phase should focus on explicit failure states, approval consistency,
policy enforcement, replay completeness, evidence freshness, agent handoff
safety, and release smoke hardening.

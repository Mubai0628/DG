## Current Capabilities

- Convert remains the real `web_table_to_csv` conversion flow.
- Approved apply / rollback remains human-approved, bounded, and rollbackable.
- Git/shell verification safe lanes remain fixed, bounded, and summary-only.
- Production Memory / Project Knowledge remains summary-only and replayable.
- MCP read-only connection remains fixed-profile and policy-gated.
- MCP read-only tool execution remains approved, bounded, read-only, and
  summary-only.
- Plugin/skill metadata governance remains descriptor-first and non-executing.
- Fixed multi-agent execution remains static and summary-only.
- Desktop Observer remains metadata-only.
- Desktop Action Proposal remains proposal-first.
- Approved narrow and expanded desktop actions remain bounded and
  human-approved.
- Cross-surface workflow remains available through controlled lanes.
- External capability execution hardening now checks policy/lease consistency,
  MCP read-only consistency, plugin/skill sandbox signals, replay completeness,
  redaction, App read-only audit summaries, and golden safety smoke.

## Disabled Capabilities

- Broad desktop automation is disabled.
- Remote control is disabled.
- Screen recording is disabled.
- Hidden background control is disabled.
- Clipboard write by default is disabled.
- File dialog automation by default is disabled.
- Arbitrary native bridge is disabled.
- Arbitrary Git/shell execution is disabled.
- Mutating MCP tools are disabled.
- Arbitrary plugin/skill runtime execution is disabled.
- Broad PermissionLease issuance is disabled.

## Known Warnings

- GitHub Actions reports the known Node.js 20 deprecation annotation for
  actions forced onto Node.js 24.
- App build keeps the known Tauri bundle identifier and Vite chunk-size
  warnings.

## Release Hygiene

- Events and replay remain summary-only.
- Raw source, raw diff, raw CSV, raw prompt, raw response, raw tool output,
  raw package metadata, stdout/stderr, raw screenshot, raw OCR, raw target
  text, and API keys are not allowed in events, replay, telemetry, or App
  surfaces.
- Conformance and eval artifacts remain generated artifacts and are not
  committed.

## Follow-up

P1I should harden desktop operator recovery and action safety without widening
desktop action capability. The next phase should focus on mismatch recovery,
stale target blockers, interruption/focus loss handling, undo and compensating
action summaries, read-only App recovery surfaces, replay completeness,
privacy/redaction audit, golden smoke, and manual QA.

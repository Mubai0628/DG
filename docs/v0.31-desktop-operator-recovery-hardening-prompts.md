
# DW-P1I-001 — Desktop Operator Recovery ADR / Threat Model / Implementation Gate

## 任务目标

建立 Desktop Operator Recovery / Action Hardening 的 ADR、威胁模型和实现门禁。

## 范围

Docs / design only。

禁止新增执行能力。

## 文件变更

新增：

- `docs/adr/0018-desktop-operator-recovery-hardening.md`

更新：

- `docs/README.md`
- `app/test/desktop-shell.test.ts` docs-lock

## ADR 必须决定

- Recovery hardening 不扩大 action set。
- v0.31 只处理 mismatch / stale / interrupted / focus loss / undo summary / replay completeness。
- Desktop action 仍需 approval receipt + typed confirmation。
- Desktop action replay 仍不能 re-execute。
- Recovery output 必须 summary-only。
- Raw screenshot / OCR / target text 不得持久化。
- App surface 只读或 recovery summary，不触发任意 desktop action。

## Threat model 必须覆盖

- stale screen
- stale window target
- focus loss
- window moved / closed / title changed
- display topology changed
- action interrupted
- partial success ambiguity
- simulated success but real no-op
- wrong target activation
- privacy leak via screen/window metadata
- raw screenshot/OCR leakage
- replay re-execution risk
- undo mismatch
- compensating action overreach
- agent desktop control bypass
- native bridge expansion risk

## Implementation gate 必须可测试

Gate categories:

- observation freshness
- target identity
- focus/window consistency
- interruption detection
- undo/compensation summary
- replay completeness
- privacy/redaction
- App UI disabled/explicit boundaries
- CI/boundary checks

## 通用 Git 工作流

每个子任务开始时运行：

```powershell
git status --short
git status -sb
git log --oneline origin/main..HEAD
```

如果发现与当前子任务无关的 dirty changes，停止并报告。

每个普通子任务结束时：

```powershell
git diff --check
git diff --cached --check
```

每个普通子任务必须本地 commit。

普通子任务禁止：

```text
git push
git tag
gh release create
```

最终 RC 子任务才允许 push / tag / release。

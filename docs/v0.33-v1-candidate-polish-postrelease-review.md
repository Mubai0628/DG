- No Git commit or Git push from the App.
- No autonomous loop.
- No raw output persistence.
- No broad native bridge.
- No mutating MCP tools.
- No arbitrary plugin or skill runtime.
- No arbitrary desktop automation.
- No cloud sync or telemetry upload.
- No destructive migration or silent deletion.

## Release Hygiene

- Release docs and summaries must not persist raw prompt, raw source, raw diff,
  raw model response, reasoning content, raw CSV, raw DOM, screenshot content,
  API keys, Authorization values, or raw shell output.
- Generated artifacts remain ignored: `app/dist`, `app/src-tauri/target`,
  `runtime/dist`, `browser-extension/dist`, `conformance/results`, `.tmp`,
  logs, coverage, screenshots, and release zips.
- Conformance and eval artifacts remain local diagnostics and are not release
  payloads by default.

## P1L Handoff

P1L should establish a permission mode and execution policy foundation for
future high-privilege automation without opening that authority in v0.34. The
next phase should model permission modes, session leases, capability gates,
risk budgets, kill switch state, App previews, and summary-only audit/replay.

## 2. 工作方式

每个普通小阶段必须：

1. 检查 git 状态。
2. 只改本阶段相关文件。
3. 运行 scoped checks。
4. 本地 commit。
5. 不 push。
6. 不 tag。
7. 不创建 GitHub Release。

最终 RC 小阶段才允许：

1. full gates。
2. push main。
3. tag。
4. push tag。
5. 创建 GitHub pre-release。

## 3. Git 工作流


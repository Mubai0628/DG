## Current Capabilities

- Permission mode metadata for approval, autonomous safe, advanced workspace,
  full access, and break-glass modes.
- Session lease schema with time, workspace, scope, and revocation summaries.
- Execution policy decision model for capability-to-mode gates.
- Risk budget summaries.
- Kill switch and scoped session control summaries.
- Permission mode audit and replay summaries.

## Disabled Capabilities

- Arbitrary shell.
- Automatic command execution.
- Automatic apply.
- Arbitrary file deletion.
- Recursive delete.
- Git commit or push.
- Autonomous loop.
- Full access execution.
- Raw transcript persistence.

- No arbitrary shell or command broker was enabled.
- No automatic apply or rollback path was added.
- No recursive delete path was added.
- No Git write path was added.
- No autonomous loop or full access execution path was added.
- No raw transcript persistence was added.
- No API key, raw prompt, raw response, raw command output, or raw source was
  introduced as a persisted artifact.

## Boundary Lock

P1L establishes permission and policy metadata only. It prepares the project for
future high-privilege work without opening high-privilege execution.

P1M may add raw transcript and output persistence infrastructure, but it must be
redacted by default, raw opt-in only, and must not add arbitrary shell, command
execution, auto apply, recursive delete, Git commit or push, autonomous loops,
or full access execution.

# v0.34 Permission Mode / Execution Policy Foundation 阶段提示词

任务目标：完成所有小阶段任务。

本阶段目标是为后续“类似 Codex / Claude Code 的高权限自动化、loop 工作、Advanced / Full Access 模式”建立统一权限控制地基。v0.34 **不直接开放**任意 shell、自动 apply、递归删除、Git commit/push、自主 loop 或 raw output 持久化；本阶段只实现 permission mode、session lease、execution policy、capability-to-mode gate、risk budget、kill switch state、App mode switch 预览和 summary-only audit/replay。

---

## 0. 背景与总目标

当前已完成到：

```text
v0.33.0-v1-candidate-polish-rc.1
```

当前系统已经具备：

```text
DeepSeek live proposal generation
App-side approved apply / rollback
Git / shell safe verification lanes
Project Knowledge / Memory
MCP read-only connection and read-only tool execution
Plugin / Skill sandbox governance
Fixed multi-agent execution
Desktop Observer
Desktop Action Proposal
Approved Desktop Action Execution
Approved expanded click/type lane
Cross-surface workflow
Replay / Audit / Redaction / Recovery
Packaging / Migration / QA / v1 candidate readiness
```

用户下一目标：设计并逐步实现类似 Codex / Claude Code 的高权限自动化模式，通过权限等级控制：

```text
审批模式
自主模式
高级模式
Full Access 模式
```

最终支持：

```text
任意 shell
自动执行命令
自动 apply
删除任意文件
删除目录 / 递归删除
Git commit / push
自主连续执行
raw output 持久化
```

但 v0.34 只做基础权限和策略框架，不开放这些高风险能力。

## 1. 本阶段高层实现方案

### 1.1 目标权限等级

本阶段需要定义并实现以下模式的 schema、policy、readiness、UI 表达和 audit/replay，但不要实际扩大执行能力：

```text
L0 Read-only / Preview
L1 Approval Mode
L2 Autonomous Safe Mode
L3 Advanced Workspace Mode
L4 Full Access Mode
L5 Break-glass Mode, design-only / disabled
```

建议解释：

| Level | 名称                    | 当前 v0.34 行为                                           |
| ----- | ----------------------- | --------------------------------------------------------- |
| L0    | Read-only / Preview     | 只读、预览、summary-only                                  |
| L1    | Approval Mode           | 当前默认模式，所有写入和高风险执行仍需人工审批            |
| L2    | Autonomous Safe Mode    | 只建立策略，暂不启用自动 loop；未来可自动低风险验证和修复 |
| L3    | Advanced Workspace Mode | 只建立策略，暂不开放 arbitrary shell / auto apply         |
| L4    | Full Access Mode        | 只建立策略和危险提示，暂不开放 full access                |
| L5    | Break-glass Mode        | design-only，默认隐藏 / 禁用                              |

### 1.2 本阶段必须输出的核心模块

```text
runtime/src/execution/permission-modes/
  mode-policy.ts
  session-lease.ts
  execution-policy-engine.ts

  index.ts

app/src/execution-policy-view.ts
app/src/permission-audit-view.ts
```

```text
pure TypeScript helpers

no broad execution
no shell expansion
no file deletion expansion
no Git write expansion
no App live automation expansion
```

### 1.3 本阶段之后的路线，仅文档锁定

v0.34 后推荐阶段：

```text
v0.35 Raw Transcript / Output Persistence
```

# v0.35 Raw Transcript / Output Persistence Post-Release Review

This review locks the v0.35 raw transcript and output persistence release line.
P1M is complete.

P1M should not continue expanding execution authority. The next phase is P1N:
Arbitrary Shell / Command Broker.

## Release

- Tag: `v0.35.0-raw-transcript-output-persistence-rc.1`
- Title: Raw transcript and output persistence foundation, no arbitrary shell
- Status: released pre-release candidate

## Checks

- Scoped checks: passed during the P1M ordinary tasks.
- Stage-end full gates: passed during the P1M-009 RC polish.
- Manual GUI QA: tracked by the v0.35 manual QA checklist.
- GitHub Actions: main and tag checks passed for the v0.35 release commit.

## Current Capabilities

- Transcript schema.
- Redacted output contract.
- Transcript store commands.
- App transcript viewer.
- Retention, export, and delete policy.
- Transcript replay and audit.

## Disabled Capabilities

- Arbitrary shell.
- Automatic command execution.
- Auto apply.
- Recursive delete.
- Git commit or push.
- Autonomous loop.
- Full access execution.

## Known Warnings

- Existing Tauri bundle identifier warning is non-blocking if gates pass.
- Existing Vite chunk-size warning is non-blocking if gates pass.
- GitHub Actions Node 20 deprecation annotation is non-blocking if checks pass.

## Release Hygiene

- Transcript records are redacted by default.
- Raw output remains opt-in only.
- No arbitrary shell or command broker was enabled.
- No automatic command execution was added.
- No auto apply, recursive delete, Git commit or push, autonomous loop, or full
  access execution path was added.
- Command-like output must stay summary-only unless future policy explicitly
  permits raw transcript retention.

## Boundary Lock

v0.35 establishes the transcript persistence foundation only. It prepares the
project for future command broker work, but it does not authorize command
execution.

P1N may introduce a permission-mode-gated command broker, but command output
must enter the transcript pipeline, replay must stay summary-only, and high-risk
execution remains denied unless later gates explicitly allow it.

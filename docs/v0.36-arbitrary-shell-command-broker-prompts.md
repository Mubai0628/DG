## 1. 阶段总目标

实现一个受 Permission Mode 管控的命令执行中枢：

```text
permission mode / session lease
→ command request
→ policy decision
→ dangerous command classification
→ transcript capture
→ bounded runtime execution
→ kill switch / timeout
→ redaction
→ summary event / replay
→ App gated command surface
```

本阶段允许在明确模式下打开：

- Approval Mode：仍只允许既有 fixed safe verification lanes。
- Autonomous Safe Mode：允许 allowlisted verification commands 自动执行。
- Advanced Workspace Mode：允许 workspace-scoped arbitrary shell，仍需要 scope / lease / risk budget。
- Full Access Mode：可以建模为最高权限，但本阶段默认不启用 full access broad execution；如果实现，也必须 disabled by default / explicit lease only。

本阶段不得实现：

- auto apply
- recursive delete
- Git commit / push
- autonomous loop
- full access session end-to-end
- broad native bridge
- desktop broad automation
- cloud upload
- telemetry upload

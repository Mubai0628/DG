## Boundary Lock

The only allowed controlled-creation side effect remains explicit
user-triggered Record Draft Event, which appends a summary-only local draft
event. All validation, approval, virtual apply, rollback, and replay surfaces
remain validation-only, audit-preview, draft-only, in-memory-summary-only,
checkpoint-preview, replay-preview, read-only, or planning-only.

Future P0J work must preserve:

1. patch-first design,
2. approval-first design,
3. rollback-first design,
4. sandbox or disposable workspace isolation before any real apply,
5. no direct user workspace mutation by default,
6. no raw prompt, raw source code, raw diff, raw DOM, raw CSV, API key,
   authorization header, environment value, screenshot, clipboard content, or
   full memory content in event payloads or preview surfaces.

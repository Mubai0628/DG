## Boundary Lock

P0K proves that user workspace apply and rollback can be modeled as
runtime-only explicit fixture-root prototypes with summary-only event writing.
It does not authorize App-side apply, rollback, approval execution, event
writing, Git, shell, native bridge, or desktop action.

P0L must move toward model-assisted proposal generation only. DeepSeek may
produce structured patch proposal drafts in a later gated task, but those
drafts must enter the existing validation, audit, approval, virtual apply,
rollback, event, and replay chain before any future execution milestone can be
considered.

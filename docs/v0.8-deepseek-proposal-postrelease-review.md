## Boundary Lock

P0L proves that model proposal drafts can be represented, repaired, imported,
and projected through the existing validation, audit, approval, rollback, and
replay preview chain without live DeepSeek calls. It does not authorize API key
reads, fetch/network, App model calls, App execution, apply, rollback,
EventStore writes, Git, shell, native bridge, or desktop action.

P0M may design an explicit opt-in live DeepSeek proposal adapter, but live
model output must remain draft-only and must enter the P0L schema, repair,
validation, audit, approval, virtual apply, rollback checkpoint, and replay
chain before any later execution milestone can be considered.

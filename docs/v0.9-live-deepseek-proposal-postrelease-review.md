## Boundary Lock

P0M proves that an explicit opt-in runtime live proposal adapter can exist
behind injected resolver and injected transport boundaries without App live
calls, App key reads, App fetch/network, App execution, apply, rollback,
EventStore writes, Git, shell, native bridge, or desktop action.

P0N may evaluate live proposal quality and golden cases, but model output must
remain proposal-only and must enter schema, repair, validation, audit, approval,
virtual apply, rollback checkpoint, replay, and redaction-audit chains before
any later execution milestone can be considered.
